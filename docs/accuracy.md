# 精度設計 (accuracy)

このドキュメントは精度プロファイル、**誤差バジェット（トップダウン配分）**、検証戦略、非保証事項を定める。
方針は **精度最優先**。許容誤差は「テストで決める」ではなく**目標から逆算で配分**し、各層へ割り当てる。

> 状態: ドラフト（Milestone 0）。配分値は設計目標であり、Milestone 2 以降の**実測で確認・調整**する。実測前に保証値として公開しない。

---

## 0. 最重要原則: 精度を2系統に分離する

日食予報の誤差には、**計算で詰められる部分**と**原理的に予測律速の部分**がある。これを混ぜると将来日食の精度を誤って約束してしまう。

- **(a) 幾何/相対精度（TT基準）** — 継続時間・食分・中心線の相対形状・最大食の相対時刻。
  天体暦と幾何計算の質で決まり、sub-arcsec まで詰められる。**計算律速**。
  - 脚注（局地接触のμ依存）: 純TT として扱えるのは **地心ベッセル要素 (x, y, d, l1, l2) と全球gamma・最大食「時刻TT」**（影軸幾何のみ）。一方、**局地接触時刻の幾何分は見かけ時角 μ を経由するため δUT1 に依存する**（μ = ERA(UT1) − α_z、conventions §5/§6）。よって局地接触時刻は (a) に分類しつつ δUT1 混入を含む。詳細バジェットは §2.1L（局地 topocentric）参照。
- **(b) 絶対UTC精度** — 接触時刻を UTC で語る部分。
  ΔT (= TT − UT1) と UT1−UTC の予測で律速される。過去・近傍は EOP 実測で <0.1s だが、**将来（〜2100）は数秒に増大**する。**予測律速**。

→ 接触時刻は **TT と UTC の両方を返す**（conventions §6）。`CalculationMetadata` に ΔT モデルと**その不確実性帯**を必ず含める。READMEと非保証事項に明記する。

### 0.1 精度の定義: 「検証可能精度」と3層の切り分け（B5）

本書の数値目標（最大食TT ±1〜2s / 接触 ±2s / 食分 ±0.0005 / 中心線 sub-km）は、**規定モデル仮定下での「検証可能精度」**として定義する。すなわち **平均月縁・点光源・指定 ΔT/暦/k モデル**という仮定の下で、**オラクル（DE 再計算 / NASA 公開幾何）と一致する範囲**を指す。**目標 ≠ 保証 ≠ 現実観測精度**であり、これらを区別する。

数値目標は維持するが、その意味は「**オラクル比較・モデル内**」である。中心線 sub-km は「**DE 再計算 / NASA 幾何で検証可能な範囲**」を意味し、現実の sub-km 保証ではない。

精度は次の3層に整理する:

- **層1: 検証可能な幾何（計算律速・モデル内）** — 全球gamma・最大食「時刻TT」・継続時間・食分・中心線の相対形状。規定モデル仮定下でオラクル（DE 再計算 / NASA ベッセル要素表）と一致する範囲で検証可能。本書の数値目標が指すのはこの層。
- **層2: 現実観測精度（保証外）** — 実際の月縁地形・大気・観測機材を伴う現実の観測接触。**現実観測接触は月縁で ±数秒〜15s 不定**であり、平均月縁・点光源モデルの数値目標は**この層を保証しない**。
- **層3: 将来UTC（予測律速・保証外）** — 接触時刻を UTC で語る部分。ΔT/UT1 予測で律速され、将来（〜2100）は数秒に増大する。**将来UTC・1900–1961（δUT1 律速）は保証外**。

→ まとめ: **検証可能精度（層1, モデル内・オラクル比較）は目標として担保を狙うが、現実観測接触（層2）・将来UTC/1962以前（層3）は保証しない**。詳細な非保証は §6。

## 1. 精度プロファイル

`AccuracyProfile { Standard, Reference }`（**公開プロファイルは2層**）

> **内部粗スキャン（coarse scan・非公開）**: 探索段の高速化は**公開プロファイルではなく非公開の実装 detail**として持つ。**候補棄却専用**であり、**偽陰性ゼロのマージンで保証**する（§3.4 / §検索戦略の D6）。簡易太陽暦・簡易月暦・簡易ΔT・低次補正を用い、**章動は IAU2000B を許容**してよい（公開出力は常に Standard = IAU2006/2000A）。**報告される日食/接触は必ず Standard で再計算**するため、出力は検証可能精度（§0.1）に統一される。内部粗スキャンの棄却マージン（例: 最大食概算の粗さ）は**出力精度の目標ではなく探索の正しさ要件**であり、数値目標としては扱わない。

### Standard（本番標準・厳格化後）
- 用途: 通知、一般利用、本番運用。
- モデル: **VSOP87D（太陽）+ ELP/MPP02（月）フル系列**、IAU2006歳差 + IAU2000A章動、light-time + aberration **必須ON**、UT1/TT分離、WGS84楕円体、観測者標高、EOP（UT1-UTC・極運動）適用。
- 目標（厳格化後 / TT基準の幾何精度。すべて**検証可能精度＝オラクル比較・モデル内（平均月縁・点光源・指定 ΔT/暦/k モデル）**。§0.1）:
  - 最大食時刻: **±1〜2 秒**（オラクル比較・モデル内）
  - 局地接触時刻: **±2 秒**（幾何分。オラクル比較・モデル内。UTC絶対は §0(b) 参照）
  - 食分: **±0.0005**（オラクル比較・モデル内）
  - 中心線位置: **sub-km（≲0.5 km、幾何分）**（DE 再計算 / NASA 幾何で検証可能な範囲。現実 sub-km 保証ではない、§0.1）
- 注意: UTC で報告する将来接触時刻は ΔT/UT1 予測律速。**現実観測接触（月縁地形を伴う）は ±数秒〜15s 不定で保証外**（§0.1 層2 / §6）。

### Reference
- 用途: 回帰試験、ベンチマーク、高精度再計算、**差分テストの第一義オラクル**。
- モデル: **JPL DE 系**（太陽・月の位置のみ DE から取得）、高精度時刻・地球姿勢、細かい探索間隔、厳しい収束条件。
- 日食幾何・ベッセル生成・接触計算は**自前**。→ 暦由来誤差と幾何由来誤差を分離できる。
- feature `jpl` で有効化。巨大データは crate 本体に同梱しない。

## 2. 誤差バジェット（Standard・厳格化後）

### 2.1 幾何/相対精度: 最大食時刻 目標 ±1.5 s

感度: 太陽-月の相対角速度 ≈ 0.5″/秒 → **1″ ≈ 2 s**。許容角度バジェット ≈ 0.75″（±1.5s 相当）。RSS配分:

| 層 | 配分 (RSS) | 担保手段 |
|---|---|---|
| 月 地心見かけ位置 | 0.40″ | ELP/MPP02 打切り次数で担保（下記 §2.4） |
| 太陽 地心見かけ位置 | 0.20″ | VSOP87D（Earth heliocentric を反転） |
| 光行時間 + 光行差 | 0.10″ | Standard 必須 |
| 歳差章動 + フレーム | 0.05″ | IAU2006/2000A（実力 ~1mas、余裕） |
| solver 収束 | 0.05″ | root_tolerance を目標の 1/10 以下 |
| 多項式 fit（使う場合） | <0.10″ | fit区間で残差を実測ガード（§3） |
| **合成** | **≈0.49″ ≈ 1.0 s** | 目標 ±1.5s に対し余裕あり |

#### 最大食時刻の数値手法（D2）

最大食時刻は **dm/dt = 0 の求根（Brent）を正式手法**とする。ここで m² = u² + v²（影軸からの観測者/地心の隔たりの二乗。中心線尖点での √ の不連続を避けるため最小化対象は **m²** に統一、距離 m そのものではない）。

- 距離最小化（m² の直接最小化）は **粗ブラケット用**に限る。最小化は最小近傍で残差 ~√eps に律速され、皆既帯の「平底」（dm/dt がほぼ 0 の区間）で収束が劣化するため、保証手法には使わない。
- dm/dt = 0 の符号変化でブラケットを取り Brent で詰めることで √eps 律速を回避し、root_tolerance を目標の 1/10 以下に保つ。
- 皆既帯平底・中心線尖点の退行を検出する fixture をテストに追加する（§3）。

### 2.1L 局地 topocentric 誤差バジェット: 局地接触時刻 目標 ±2 s（D1/C1/I1）

§2.1（地心：全球gamma・最大食「時刻TT」）は **影軸幾何のみ＝純TT** で閉じる。これに対し**局地接触時刻**は、地心月位置に加えて (a) 月距離残差→地平視差、(b) δUT1→μ の2経路が**追加で**効くため、別建てで配分し RSS 再合成する。

感度（共通）: 局地接触は太陽-月の相対角運動 ≈ 0.5″/秒 → **1″ ≈ 2 s**（§2.1 と同じ感度を流用）。

| 経路 | 残差源 | 角度換算 | 時刻換算 | 担保手段 |
|---|---|---|---|---|
| (0) 月地心見かけ位置 | §2.1 の月+太陽+補正分 | ≈0.49″ | ≈1.0 s | §2.1 を再利用 |
| (a) 月距離残差 → 地平視差 | δr/r（ELP/MPP02 距離級数残差） | π ≈ 3422″ · (δr/r)（地平視差 π = arcsin(Re/r) ≈ 3422″）。視差残差 δπ ≈ 3422″ · (δr/r) | δπ × 感度 | 距離級数を DE 差分で残差ガード（§2.4）。要確認: δr/r の達成上限は M2 実測 |
| (b) δUT1 → μ（見かけ時角） | δUT1（EOP実測/予測） | δUT1 1 ms ≈ 0.46″（赤道地球回転） | ≈ ~1 s / 1 ms（δUT1 1 ms ≈ 0.46″ ≈ ~1 s） | 過去/近傍は EOP <数ms。将来は予測帯を metadata 出力（§2.3） |

- **RSS 合成（局地接触）** = √( (0:≈1.0s)² + (a)² + (b)² )。(a) は δr/r を M2 で 0.1″ 級（≲~0.0003 相当, 要確認）に詰めれば時刻寄与 ≪1 s。(b) は過去/近傍（δUT1 数 ms）なら ~数 s 未満だが、δUT1 ≲1 ms 相当の近傍では ~1 s 内に収まる。→ **近傍・実測 EOP 域では局地接触 ±2 s は閉じる見込み**。将来年代は (b) が予測律速で増大する（§0(b)/§2.3、UTC絶対精度と同根）。
- 注記（純TTとの切り分け）: **全球gamma・最大食「時刻TT」は影軸幾何のみ＝純TT**（δUT1 非依存）。**局地接触時刻の幾何分のみ δUT1（→μ）を含む**（§0(a) 脚注）。地心月位置残差は両者共通だが、視差(a)・μ(b) は局地接触に固有。
- 値は設計目標。(a) の δr/r 上限、(b) の運用域別 δUT1 は **M2 実測で確定**（実測前に保証値化しない）。

### 2.1E 局地最大食時刻のエッジ限界（ゴールデン20 実測で確認）

最大食時刻 ±1〜2s は**中心線近傍〜中程度の食**で成立する。次の2エッジでは時刻のみ劣化する（**食分・食面積・高度は不変＝物理的影響は軽微**）。NASA/USNO 比較で実測：

- **極浅い部分食（観測者が軸から遠く m_min が大）**: 最大食「時刻」誤差が増大（食分 0.12 級で ~12s、部分域 ~6–8s）。原因は数値条件：`solve_local_maximum` は `m²` の中心差分で `d(m²)/dt=0` を解くが、`m²≈m_min²` の絶対値が大きいと暦評価の相対誤差 ε による差分ノイズ床 `~ε·m_min²` が信号 `~v²·δt` を埋め、求根が甘くなる（曲率 `2v²` 自体は深さ非依存だが SNR が m_min² で劣化）。**その地点では食分・食面積が平坦＝最大食時刻の不定は観測上無意味**。改善余地＝解析微分（x′,y′）供給 or m での求根（部分食は中心線尖点が無く m 直接最小化が可・要検討）。
- **日の出/日没食で幾何最大食が地平下**: 本エンジンは**幾何的**最大食（地平下でも）を返す。一方 NASA/USNO は**観測可能**域に制約した最大を報告するため、幾何 vs 観測可能の定義差で最大食時刻が ~5–13s ずれ、**可視性クラスも食い違いうる**（例: 日の出食で幾何最大が日の出前＝エンジン `BelowHorizon` / オラクル `SunriseEclipse`）。要設計判断：幾何最大を正とし可視性フラグで補足するか、観測可能最大へ切替えるか（api-draft の可視性6値・§0.1 層との整合で確定）。

### 2.2 食分・食面積

- 感度: 太陽視直径 ≈ 1920″ → **0.001食分 ≈ 1.9″**。目標 ±0.0005 → 相対位置 **≲1″**。
- acos 引数は丸め誤差を考慮し `[-1, 1]` にクランプ。境界（離隔/内包/外接/内接/同半径）を明示処理。
- 月半径 k の選択（conventions §9）で皆既/金環境界の系統差が出る。照合時は Espenak 慣習へ切替。

### 2.3 絶対UTC精度（接触時刻を UTC で報告する場合）

| 要因 | 過去/近傍 | 〜2100方向 | 性質 |
|---|---|---|---|
| ΔT (TT−UT1) | <0.1 s（IERS実測） | **数秒**（予測外挿） | 共通モード（相対形状に効かない） |
| UT1−UTC | <数 ms（EOP） | 予測ぶん増大 | 地球回転位相 |
| 極運動 → 地上 | ~10 m | 同 | 中心線位置 |
| 観測者/楕円体/標高 | sub-m | 同 | WGS84 で十分 |

- δUT1 1 ms ≈ 地球回転 0.46″（赤道）≈ 観測者 ~14 m。中心線・接触に小さく寄与。
- **将来UTCの不確実性帯を `CalculationMetadata` に出力**する。

### 2.4 ELP/MPP02 打切り次数（要 §3.3 実測で確定）

- 目標: 月 地心位置 ≲0.40″（§2.1）。
- 方針: 経度・緯度・距離の各級数を、JPL DE 差分で残差が **0.1″ 級**に収まる項数まで採用（過剰な打切りを避ける）。
- 確定手順は §3.3。打切り次数と達成残差は `metadata`（ephemeris_version）に記録。
- VSOP87D 太陽側も同様に DE 差分で残差 0.05″ 級を目標に項数決定。

## 3. 検証戦略

### 3.1 オラクルの階層（目標より良いオラクルを使う）

1. **第一義(暦層): JPL DE 差分テスト（Reference バックエンド）**
   解析暦 と DE で**同一のベッセル/接触パイプライン**を通し、差分を取る。→ 暦誤差と幾何誤差を分離（§4 層分解）。CI 必須にせず nightly / 手動ジョブ（巨大データのため）。
   - **限界（重要）**: DE 差分は解析暦と DE を**同一の自前ベッセル/接触パイプラインに通して**比較するため、**検証できるのは暦層のみ**である。幾何パイプライン（ベッセル要素生成・接触計算）に共通バグがあると、両系列に同じバグが乗るため**差分には現れず検出できない（幾何共通バグは検出不能）**。
2. **第一義(幾何層): 幾何層の独立オラクル（NASA ベッセル要素表 / 独立第二実装）**
   暦層の DE 差分とは独立に、**幾何層を第一義級で検証する**。手段は (i) **NASA 公開のベッセル要素表**と自前生成ベッセル要素 (x, y, d, l1, l2, μ) を突合、または (ii) **幾何パイプラインの独立第二実装**との突合。これにより、上記 1 の DE 差分では検出できない**幾何共通バグを捕捉**する。慣習差（ΔT・k）を明記した上で評価する。
3. **第二義: NASA / USNO 公開値（最終結果の整合チェック）**
   ΔT・k 慣習を揃えないと系統差が出るため、**慣習差を明記した上での整合チェック**に留め、絶対基準にしない。
4. **MockEphemeris**: 完全中心 / 明確な金環 / 明確な部分食 / 影が地球を外す、等の人工配置で幾何ロジックを単体検証。

### 3.2 テストレベル（plan §16 準拠）

- **L7 に追加サブテスト**: 「直接瞬時計算 vs 多項式fit」の残差を実測し、profile 毎に採用方式を決定（architecture `BesselianSource`）。

### 3.3 ELP/MPP02・VSOP87D 項数確定手順（二段ゲート: M2 暫定確定 → M10 DE確定）

**フル/Reference 版 JPL DE は M10**（036 フル版）だが、項数確定を M10 まで盲目（暫定オラクルのみ）にはしない。**036 の最小 SPK reader（DE440 の太陽/月のみ・Chebyshev type2/3 評価）を M2 必須に昇格**させ、M2 時点から DE 実値で打切り次数を決める。これにより reader 選定（自前 vs 既存）というフル版の問題を切り離しつつ、暦層の第一義オラクルを M2 から使える。項数確定は二段ゲートとする（ISSUE-047 暫定オラクル戦略を参照）:

1. **暫定確定（M2）** — フル DE 投入前だが、**M2 で最小 SPK reader（036, 太陽/月のみ）を必須**として DE 実値で打切り次数を決める。盲目にはしない:
   - **最小 SPK reader（036, M2 必須）**: DE440 の太陽/月のみを Chebyshev type2/3 で評価し、**暦層の第一義オラクルとして打切り次数を決定**する。
   - MockEphemeris（人工配置）で幾何ロジックの健全性を確認。
   - **幾何層の独立オラクル（§3.1-2, NASA ベッセル要素表 / 独立第二実装）**で幾何共通バグを検証。
   - SOFA（時刻・姿勢・見かけ位置の参照）と突合。
   - NASA/USNO 公開値を**第二義**の整合チェック（慣習差を明記、絶対基準にしない）。
   - フル係数を取り込み、項を寄与順に並べ、最小 SPK reader（DE）基準で累積残差が目標（月0.1″/太陽0.05″）を切る最小項数を採用。
2. **DE確定（M10）** — フル版 JPL DE 差分（Reference バックエンド、036 フル版投入後）で M2 項数を再検証し、達成残差を**正式確定**。必要なら項数を改訂。
3. 採用項数・達成残差・検証時刻範囲、および「暫定(M2)/確定(M10)」のいずれの段かを `metadata` と本書に記録。

#### M10 DE確定 結果（2026-06-20・段階=確定(M10)）

フル版 Reference（`JplEphemeris`, DE440s SPK, feature `jpl`, ISSUE-036・SPICE 突合で位置<10m/速度<1mm/s）を
第一義オラクルに、`AnalyticalEphemeris` の地心方向を **1900–2100・n=2435 点**で突合（`tests/de_diff.rs`）。
**打切りは行わず全項を採用**（支配誤差はフレーム/永年であり項数ではない＝full でも budget 内・性能十分）。
達成残差を確定:

| 天体 | 採用 | DE440 比 達成残差（幾何地心方向, 1900–2100 max） | 目標 |
|---|---|---|---|
| 太陽 | **VSOP87A**（黄道 J2000 直交）＋ VSOP87→FK5/ICRS 固定回転（geocentric ICRS）。黄道of date は VSOP87D。 | **0.040″** | 0.05″ ✅ |

- **太陽**: 当初の VSOP87D（黄道of date）＋ IAU2006 of-date 行列は、VSOP87 力学的分点と IAU2006 分点の
  歳差レート不整合（IAU1976 vs IAU2006 = 0.28″/century）で 0.37″ の系統誤差を生んだ。J2000 固定フレーム
  （VSOP87A＋VSOP87 固定回転）へ移し解消（ISSUE-013/035）。
- **月**: ELP2000-82B（LLR-fit）は DE440 と W1 永年差で補正前 1.69″。**清浄な ELP/MPP02 原データが入手
  不可**（一次配布元 cyrano-se.obspm.fr 停止・IMCCE/CDS/Wayback にミラー無し・GPL ytliu0 は混入禁止）の
  ため、W1 永年項を DE440 差分で最小二乗再フィットした簡易 DE-fit で 0.268″ へ低減（月 budget 0.40″ 達成）。
  残差床（黄経の周期 Poisson 増幅項＋緯度系統差 ~0.13″）は ELP2000-82B の限界で、**≤0.1″ の理想は
  ELP/MPP02 入手まで残課題**（ISSUE-014/034）。

## 4. 誤差の層分解（レビューゲート §25）

天体位置が不正確なまま日食側に補正を足して隠さない。誤差は必ず以下に分解して報告:

- 時刻変換誤差 / 太陽位置誤差 / 月位置誤差 / 影幾何誤差 / 多項式近似誤差 / 接触solver誤差。

JPL DE 差分（§3.1）で暦誤差を切り出し、残りを幾何/数値層へ帰属させる。

### 4.1 実装した層分解の粒度（ISSUE-030 `report_differential`・2026-06-20）

上記 6 物理層は理想だが、**shadow_geometry / polynomial_fit / contact_solver の個別分離はエンジン内部の
計装を要し ISSUE-030 のスコープ外**である。`umbra-fixtures::report_differential`（DE 差分・誤差層分解）は、
**同一のベッセル/接触パイプラインに解析暦と JPL DE を通す 2 エンジン差分法**で清浄に切り出せる粒度＝
**2 バケットへ集約**して実装した（ISSUE-030 §38-45 スケッチからの確定逸脱・本節に記録）:

- **暦層 `ephemeris` = analytical − DE**（同一パイプライン → 差は暦のみ）。§2.1 の太陽/月位置誤差に対応。
- **幾何/数値層 `geometry` = DE − golden オラクル**（DE 入力 → 残差は影幾何＋多項式＋接触 solver＋慣習差の統合）。
- **総合 `total` = analytical − golden**。各 metric・各サンプルで **符号付き `total ≡ ephemeris + geometry`**。

各 metric（全球: 最大食時刻/食分、地点: 最大食接触時刻/C1〜C4 接触/食分/食面積/高度）を 3 層に分解する。
**誤差を隠さない原則**（conventions §11）は「内部の影/多項式/接触層を個別測定せず幾何層に統合表示する」と
明記して担保する。なお**太陽/月の生の方向残差**（太陽 0.040″ / 月 0.268″・§3.3）は `umbra-ephemeris` の
DE 差分（`tests/de_diff.rs`）が暦層で別途確定済みで、本レポートはそれを**日食 metric への寄与**として
再表現する位置づけ。合否判定（`ToleranceProfile`）は持たない純粋な誤差層分解（原因層の特定が目的）。

## 5. ΔT / UT1 / EOP データ

- 履歴: IERS EOP 14 C04（UT1−UTC、極運動）。
- 長期ΔT: Espenak–Meeus 多項式（1972以前・外挿）。
- 将来: IERS Bulletin A 予測（短期）＋以降は外挿。**不確実性帯を併記**。
- データはバージョン + checksum 管理（plan §8 / conventions §6）。**実行時ネットワーク禁止**、更新は xtask に隔離。

## 6. 非保証事項（README にも明記）

- 研究・測地・航法・安全上重要な用途には使用しない。
- **数値目標は「検証可能精度」＝オラクル比較・モデル内（平均月縁・点光源・指定 ΔT/暦/k モデル）であり、目標 ≠ 保証 ≠ 現実観測精度**（§0.1）。
- 月縁地形（ベイリービーズ、厳密 C2/C3 の山谷補正）は考慮しない。**現実観測接触は月縁地形により ±数秒〜15s 不定であり、平均月縁・点光源モデルの数値目標は現実観測接触を保証しない**（§0.1 層2）。
- 中心線 sub-km は DE 再計算 / NASA 幾何で**検証可能な範囲**を指し、**現実の sub-km は保証しない**（§0.1）。
- 地平線付近の厳密な大気差、観測機材固有条件、雲量・天候は対象外。
- 月食・惑星食・恒星掩蔽は v1.0 対象外。
- **将来・過去年代では ΔT/UT1 予測により UTC 絶対時刻の誤差が増大する**（§0, §2.3）。**将来UTC は予測律速で保証外。1962 以前は EOP（IERS）実測前のため δUT1 律速で保証外**（δUT1 1 ms ≈ 0.46″ ≈ ~1 s、§2.1L(b)）。
- v1.0 初期対応年代は 1900〜2100。
