### 循環チェック
層順（core → time → ephemeris/frames → eclipse-geometry → bessel → global/local → engine → fixtures/cli → strategy）で並べると、全辺が**前方（下流）向き**で後方辺が無い。
- 026 → 027（**一方向化済み**）: 旧依存行は双方向記述（026「最大食時点で 027 を評価」／027「最大食時点で本式を評価＝026 の時刻を使う」）だったが、実体は「026 が時刻を求め、その時刻で 027 を 1 回評価」する**呼出し順序**。**ビルド依存は 026 → 027 に一本化**（027 の依存から 026 を削除済み, M7 表）。循環なし。
- 022 ↔ 037 / 023 ↔ 037,022: 037 は 022/023 に依存しない、022 は 023 に依存しない（一方向）。循環なし。
- 043 は多数を集約する終端ノード（031/032 CLI のみが上流から呼ぶ。実質シンク）。044 は基盤エラーを集約し 042/043/045 へ供給（基盤側から見れば下流向き一方向）。循環なし。

→ **結論: 主要依存辺に循環は無い（DAG）。** 唯一の注意は 026↔027 の依存行双方向記述（実体は呼出し順序であり循環ではない）。
