# 規約 (conventions)

このドキュメントは `umbra-rs` 全体で固定する単位・座標系・時刻系・符号・物理定数の規約を定める。
**実装開始前に確定させ、各数式の実装コメントから本書を参照する。** 例外を作る場合は本書に追記してから行う。

> 状態: ドラフト（Milestone 0）。数値の最終確定はレビューゲート（仕様レビュー→数式レビュー）を通すこと。

---

## 1. 単位

| 量 | 内部表現 | 公開入出力 | 備考 |
|---|---|---|---|
| 角度 | ラジアン | 度も提供 | newtype `Radians` / `Degrees` |
| 距離（幾何計算） | km | km | `Kilometers` |
| 距離（観測者標高） | m | m | `Meters` |
| 距離（天体暦内部） | AU 許容 | — | `AstronomicalUnits`、境界で km へ変換 |
| 時間間隔 | 秒 (SI) | 秒 | |
| ベッセル要素 x, y, l1, l2 | 地球赤道半径 (Re) で無次元化 | 無次元 | §5 参照 |

- 公開APIで生の `f64` を「単位付き量」として渡さない（§5 型設計 / architecture.md）。
- 単位変換は境界の変換関数に集約し、計算本体に散らさない。

## 2. 角度

- 内部はラジアン。公開入出力では度も提供する。
- 正規化の既定:
  - 経度・時角など循環量: `[-π, π)`（度では `[-180°, 180°)`）。
  - 赤経・恒星時など「0〜2π で語る量」: `[0, 2π)`。**用途ごとに正規化関数を分け、混在させない。**
- 連続性が必要な求解（合・接触のゼロ点）では、±π 折返しを除いた**連続関数**へ変換してから解く（algorithms 参照）。

## 3. 経度・緯度

- **経度は東経を正**。範囲 `[-180°, 180°)`。西経入力はCLI・変換関数で吸収し、内部に西経正を持ち込まない。
- 緯度の**入力は測地緯度** (geodetic)。地心緯度が必要な箇所は内部で明示変換し、変数名で区別する（`geodetic_lat` / `geocentric_lat`）。

## 4. 距離・地球モデル

- 幾何計算は km。天体暦内部は AU を許容し、パイプライン境界で km へ。
- 地球楕円体は **WGS84** を既定（`EarthModel::Wgs84`）。
  - a = 6 378 137.0 m, 1/f = 298.257223563。
  - 赤道半径 Re（ベッセル無次元化に使う基準）= WGS84 の a を採用し、本書に固定する。NASA 慣習との差異は accuracy.md に記録する。
- 観測者標高は楕円体高 (ellipsoidal height) を既定とする。正高（ジオイド高）入力を許す場合は変換を別途明示。

### 4.1 物理定数表（固定値・041連動）

実装中の **magic number はこの表を参照**し、コードには本表への参照と出典を必ず添える（§10/§11）。値はここで一元管理し、散在させない。

| 定数 | 記号 | 値 | 出典 | 備考 |
|---|---|---|---|---|
| 光速（真空） | c | 299 792.458 km/s | IAU（SI定義値） | 光行時間・光行差。定義値（無誤差） |
| 天文単位 | AU | 149 597 870.7 km | IAU 2012 (B2) | AU↔km 変換。定義値 |
| TT − TAI | — | 32.184 s | IAU 1991（定義） | 定数オフセット。閏秒は UTC↔TAI 側で別途 |
| 地球赤道半径（WGS84 長半径） | Re = a | 6 378 137.0 m | WGS84 | ベッセル無次元化の基準（§1/§4）。NASA慣習との差は accuracy.md に記録 |
| 地球自転角速度 | ω⊕ = dERA/dt | 2π·1.00273781191135448 / 86400 rad/s（≈7.2921151467e-5） | IERS / SOFA `iauEra00` | 局地ベッセル射影の観測者速度（§5.2/ISSUE-024）。ERA の時間係数の解析微分。定義式由来 |

> 上記はいずれも定義値または採用固定値。EOP（UT1−UTC, 極運動）・ΔT・閏秒のような**時変・データ駆動の量はここに含めず** EOP/ΔT データ経由で供給する（§6・accuracy.md §5）。GM 等の追加定数が必要になった場合は IAU2015/IERS の一次資料を確認のうえ本表に追記する（**要確認**）。

## 5. 座標フレーム

右手系で統一する。実装で扱う主なフレーム:

| フレーム | 意味 | 主な用途 |
|---|---|---|
| ICRS | 国際天文基準座標系（慣性） | 天体暦の基準 |
| GCRS | 地心 ICRS 軸 | 地心見かけ位置の出発点 |
| CIRS | Celestial Intermediate Ref. System（CIO基準・of date） | ERA 適用前 |
| TIRS | Terrestrial Intermediate Ref. System | ERA 適用後・極運動前 |
| ITRS | 地球固定（≈ WGS84 軸） | 観測者・地上経路 |
| FundamentalPlane | 月影軸を z とするベッセル基本面 | 日食幾何 |

- **フレーム変換連鎖（Standard 必須）**:
  `GCRS →[frame bias + 歳差(IAU2006) + 章動(IAU2000A)]→ CIRS →[ERA(UT1)]→ TIRS →[極運動]→ ITRS`
- 型でフレームを区別する（`Position<F: ReferenceFrame>`）。型が重すぎる箇所は内部限定にしてよいが、**変数名・構造体名で座標系が判別できることを必須**とする。

### 5.1 光行差・光行時間の適用順序（D3/I3、SOFA iauAtciq 準拠）

見かけ地心位置の補正は **SOFA `iauAtciq` の順序に固定**する。すなわち **GCRS 内で次の順**:

1. light deflection（相対論的偏向。Standard では太陽による偏向のみ、`relativistic_deflection` で切替）
2. annual aberration（年周光行差）
3. その後に **frame bias + 歳差(IAU2006) + 章動(IAU2000A)** を適用して CIRS へ。

→ 「章動後に aberration」を**禁止**（順序を逆にしない）。光行時間（light-time）は位置取得時に解き、上記偏向・光行差はその幾何位置に対して GCRS 内で適用する。

### 5.2 時角・μ・影軸赤経の基準（D4/I5、ERA/CIO 統一）

- 見かけ時角・μ（= 見かけ時角）は **ERA（`iauEra00`）経由の CIO ベース**で構成する。`μ = ERA(UT1) − α_z`（α_z は影軸の赤経）。
- **分点ベース GST（`iauGst06a` 等）は Standard では使わない**（CIO/分点の二重定義を排し、ERA 経由 CIO に統一）。
- 影軸赤経 **α_z も CIRS 基準**（CIO 一貫）で求め、μ 構成と同一基準に揃える。
- μ が UT1（→δUT1）依存である帰結は accuracy.md §0(a) 脚注・§2.1L（局地接触バジェット）を参照。

## 6. 時刻系

| 時刻系 | 用途 |
|---|---|
| TT | 天体位置計算の標準（ベッセル要素もTT基準で生成） |
| TDB | Reference バックエンド（JPL DE）用 |
| UT1 | 地球回転（ERA, 恒星時） |
| TAI | 中間 |
| UTC | 公開入出力（既定） |

- 関数シグネチャに生 `f64` の JD を渡さない。`fn solar_position(time: TtInstant)` のように時刻系を型で持つ。
- **接触時刻は UTC と TT の両方を返せる**ようにする。将来日食の UTC は ΔT/UT1 予測で律速されるため（accuracy.md §絶対時刻）、TT基準の値を一級で保持する。
- ΔT = TT − UT1。UT1 − UTC（および極運動）は EOP データから取得する。閏秒は UTC↔TAI 変換で扱う。
- Julian Date は2要素表現 `JulianDate2 { part1, part2 }`（巨大JDと微小差をひとつの f64 に押し込まない）。

## 7. 方位・高度

- 方位角: **北 = 0°、東回り**（北→東→南→西）。
- 高度: 既定は**幾何学的高度**。大気差補正は `RefractionModel`（`None` / `Standard`）で分離し、通知用途では補正前後の両方を返せるようにする。
- 位置角 (position angle): 天の北を 0°、東回り（接触点の向き表現）。

## 8. 接触の定義

| 接触 | 定義 | 幾何条件 |
|---|---|---|
| C1 / C4（部分食 開始/終了） | 外接 | 中心間距離 = 太陽視半径 + 月視半径 |
| C2 / C3（中心食 開始/終了） | 内接 | 中心間距離 = \|月視半径 − 太陽視半径\| |
| 最大食 | 中心間距離 最小 | 距離関数の最小化 |

- 実装はベッセル面上の影半径と観測地点座標で解く（§接触関数は algorithms 参照）。
- 全球 P1/U1/U4/P4 はそれぞれ「地球上で部分食開始 / 中心食開始 / 中心食終了 / 部分食終了」。

## 9. 半径モデル（食分・継続時間に直結）

`conventions` で値と出典を固定し、`metadata` に必ず載せる。

### 月半径係数 k

`LunarRadiusModel`:
- **`IauMean` … k = 0.2725076**（既定）。月平均半径 / 地球赤道半径。
- **`EspenakUmbral` … k = 0.272281**（皆既・金環の本影判定。Espenak/NASA 慣習）。
- **`EspenakPenumbral` … k = 0.2725076**（部分・半影）。

> NASA/Espenak の公開カタログと照合するときは Espenak の2値慣習へ切り替える。既定（IauMean 単一値）との系統差は accuracy.md に記録する。皆既/金環の境界判定はこの選択で結果が変わりうる。

### 太陽半径

- IAU2015 公称太陽半径 **R_sun = 696 000 km**。各時刻の距離から視半径を算出。
- `SolarRadiusModel` で値と出典を保持（将来の改訂に備える）。限縁減光・実視半径補正は v1.0 では入れない（非保証事項）。

## 10. 符号・規約のメモ運用

各数式の実装コメントに必ず残す（§AI実装ルール）:
- 出典（文献・章・節・式番号）
- 入力単位 / 出力単位
- 座標系 / 時刻系
- 採用した符号規約
- 元式から変更した点（打切り次数・近似の省略など）

## 11. 禁止事項（規約レベル）

- 出典不明の数式 / magic number。
- 生 `f64` による時刻系の混在。
- 無条件 Newton 法（ブラケットの取れる Brent 法を標準とする）。
- テスト基準値を実装側へコピー / オラクル結果のハードコード。
- 許容誤差を「通すためだけ」に拡大すること。
- 天体位置が不正確な状態で、日食側に補正を足して誤差を隠すこと（誤差は層ごとに分解する。accuracy.md）。
