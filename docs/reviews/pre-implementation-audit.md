# 実装着手前 独立監査（2026-06-14）

設計チームの結論を渡さず、4観点の独立監査人が「設計は実装に耐えるか／精度は保証できるか」を反証する立場で検証。
**全4観点が「条件付きGO」**（NO-GOなし／無条件GOなし）。骨格は健全で精度（幾何/TT）は達成可能だが、着手前にM0で潰すべき矛盾と、精度主張の射程限定が必要。

監査観点: ①天体力学・誤差バジェット ②数値解析 ③ソフト設計・実装可能性 ④検証可能性(V&V)。

---

## 最重要の結論: 「精度は保証できるか」への正直な答え

精度は**3層に分けて**しか語れない（複数監査が独立に同じ結論）:

1. **幾何/モデル内精度（TT基準・平均月縁・点光源）= 達成可能かつ証明可能**（条件付き）。VSOP87D/ELP-MPP02フルは1900-2100で目標(0.40″/0.20″)に対し過剰なほど高精度。最大食時刻TT・gamma・食分の幾何分は ±1-2s/±0.0005 で閉じる。
2. **現実の観測接触時刻に対する精度 = 保証不可**。月縁地形（Watts誤差〜地形±3″≈±6km）で実接触は数秒〜極端15s不定。オラクル(NASA path edge 1-2km / USNO limb補正なし)が目標(sub-km/±2s)より粗い。→ 証明できるのは「同一物理仮定下のモデル一致」まで。
3. **将来UTC絶対時刻 = 予測律速**（ΔT/UT1、〜2100で数秒）。+ 1900-1961もEOP実測前でδUT1律速。

→ **「目標 ≠ 保証 ≠ 現実観測精度」を明文化**し、目標を「平均月縁・点光源・指定ΔTモデル下の幾何精度」と射程限定すれば、縮小された主張は証明可能。無条件の精度保証はできない。

## 着手前ブロッカー（M0で解消・収束指摘）

### B1. l2 符号の内部矛盾【critical, 確信度高】（天体力学C-1）
- `05-shadow-cone`(C6)・`08-global`(§B・fixture) は「l2<0=金環」、`06`/§0/`09`/`physical-models` は「l2<0=皆既」。**逆**。
- 最悪: `08`のMockEphemeris fixtureが誤符号→誤りをテストが追認。
- 対応: **正本=l2<0=皆既**に統一。`05`/`08`を書換え、符号規約テスト＋fixtureを§0基準で再作成。

### B2. 局地計算の既定 直接 vs 多項式の矛盾＋多項式ゲート未証明【critical, 確信度高】（数値Critical-3, ソフトC1）
- 今セッションの改訂(operations §D1: 局地=多項式)が `ISSUE-043`/`algorithms 06/07/09` の「局地=直接」と正面衝突。
- かつ <0.01″ ゲートを~6h窓で満たせる保証なし（NASA多項式はkm級・cubic±3h）。性能のため精度方針を未証明のまま上書きしている。
- 対応: **既定を直接(037)へ戻す**（精度最優先）。多項式は「M2でグリッド実測(実食×fit区間×次数)し<0.01″達成を証明してから採用するオプション」に格下げ。operations/architecture/numerical-policy/api-draft/043を整合。

### B3. TimeData/TimeScales の型・Result・crate帰属の三重不整合【critical, 確信度高】（ソフトC2/C3）
- api-draft(TimeData集約・非Result) ⇔ ISSUE-042(TimeData+TimeScales両立・Result) ⇔ ISSUE-043(TimeScales保持)。crate帰属(core or ephemeris)未確定、no_std core目標と緊張。
- 対応: 型を一意確定（集約形＋valid_to超過はResult）。`bundled()`付きデータは ephemeris/専用crateへ、coreは trait/純粋型のみ。

### B4. 検証の鶏卵＋M2項数確定オラクル不在【critical, 確信度中〜高】（ソフトC4, V&V C-1/C-3, 天体力学I-2）
- JPL DE差分は暦層しか検証せず**自前幾何の共通モードバグを検出不能**。M2はSOFA(=暦オラクルでない)・Mock(精度外)・NASA(慣習差)しかなく**0.1″の打切り次数を確定できない**→M10で覆る手戻り。
- VSOP87Dは**地球中心版D**を使うこと（EMB主系列だと6.4″/月。反転テストでは検出不可）。
- 対応: (a)`ISSUE-036`最小SPK reader(DE440の太陽月のみ)を**M2へ昇格**し打切り次数決定に使う。(b)**幾何層の独立オラクル**（独立第二実装 or NASAベッセル要素表）をaccuracy §3.1に第一義級で追加。(c)033にD版・地球中心の明示チェック。

### B5. 精度主張の射程限定【critical(証明可否), 確信度高】（V&V C-2/I-4）
- 目標を「平均月縁・点光源・指定ΔTモデル下の幾何精度」と明記。現実観測接触は月縁で±数秒不定をREADME/accuracy §1/§6に明示。EOP実測域(≈1962-)とそれ以外を区別。

## 独立に確認された健全性（追認でなく再検証で確認）
- 幾何/TT精度は達成可能（VSOP87D/ELP-MPP02フルは過剰精度）。2系統分離(TT/UTC)は正しく、将来UTC・中心線sub-kmは正しく非保証化。
- 依存グラフはDAG（026↔027のdoc表記のみ）。欠番なし。
- ジェネリックEngine+StandardEngine、LocalContactSet、#[non_exhaustive]、GPL非混入＋cargo-deny、決定性+fingerprint設計は健全。
- 桁落ちを「時刻引数生成」と「ゼロ点近傍差」に局在化した方針は正しい（f64で地心側は閉じる）。

## 総合判定
**条件付きGO（全4観点一致）**。骨格・精度(幾何)は健全。B1-B5（大半がdoc/型レベル修正）をM0完了ゲートで潰し、精度主張を射程限定すれば着手可。構造的再設計は不要。土台Issue(数学・時刻・型)は条件解消と並行着手可。
