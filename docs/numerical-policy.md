# 数値実装方針 (numerical-policy)

精度最優先プロジェクトで、誤差バジェット（accuracy.md §2）を**実装レベルで閉じる**ための数値方針を確定する。
各決定は許容寄与（角度/時刻）に紐づける。algorithms.md（数式仕様）と対で参照する。

> 状態: ドラフト（Milestone 0）。閾値の一部は M2 実測で最終確定（要確認を明記）。
> 基準値: f64 eps ≈ 2.22e-16、1″ = 4.848e-6 rad、相対角速度 ≈ 0.5″/s → **1″ ≈ 2 s**。
> 配分の上限目安: 1層あたり ≲0.05″（≈0.1 s）に抑えれば §2.1 の RSS（≈0.49″≈1.0s）が閉じる。

---

## 0. 全体原則

- **桁落ちは「時刻のエポック差」と「接触ゼロ点近傍の差」に集中する**。級数和そのものは f64 で十分（§A1）。対策はこの2点に集中投下する。
- 微分は **解析微分（級数）＞ 複素ステップ（孤立平滑関数）＞ 中心差分＋Richardson 検証** の優先順。固定 magic な刻み幅は禁止（conventions §11）。
- すべての許容閾値は「目標の 1/10 以下」を既定とし、バジェット由来の数値根拠をコメントに残す。

---

## A1. 級数和の数値安定性（VSOP87D / ELP/MPP02）

**結論: 級数和は f64 昇順ナイブ和で十分（Kahan 不要）。桁落ちの本体は時刻引数 t の生成にある。** ただしこの結論は L1 実測で裏取りするまで「検証済み」とはしない（下記訂正注記）。

各項 `A·cos(B + C·t)`（および sin）の総和。

- **誤差評価（訂正注記・要 L1 実測裏取り）**: 旧記述「項数 N~10³・Σ|A|~0.05 rad・余裕 9 桁」は**過小/誤り**。実際は**月（ELP/MPP02）で N~8000–36000・Σ|A|~0.13 rad・最速の位相引数 ~7.8e6 rad（ulp(arg) ~ 1e-9 rad）**。改めて再帰和の丸め誤差を見積もると ~ √N·eps·Σ|A| ≈ √(3.6e4)·2.2e-16·0.13 ≈ **5.4e-15 rad** 級で、sub-arcsec 目標（≲1e-7 rad）に対しなお十分な余裕（~7 桁）。**結論（昇順ナイブ和で十分・Kahan 不要）は維持**するが、上記の N/Σ|A|/位相引数は概算であり **L1 実測（級数和の丸め境界回帰, §A6）で裏取りするまで「検証済み」としない**。
- **既定**: 振幅 **昇順**（小→大）で加算（コストほぼゼロで丸めを最小化）。係数は降順格納が通例なので逆順走査。
  - Reference プロファイルのみ任意で Kahan/pairwise を選択可（過剰硬化を避け既定は昇順ナイブ）。
- **位相引数 `B + C·t`**: 範囲縮約（mod 2π）を**しない**（**ただし要確認・L1 実測で再確認**）。1900–2100（|t|≲0.1 millennium）で最速引数は |C·t| ~ 7.8e6 rad に達し、ulp(arg) ≈ arg·eps ≈ 1e-9 rad。libm cos の引数縮約精度に依存するため、**非縮約のまま libm cos が ~1 ulp 精度を保つかは L1 実測で再確認する**（事前に mod 2π すると逆に丸め誤差を注入しうるため既定は非縮約だが、最速引数の大きさ的に検証が要る）。

### 時刻引数 t の生成（ここが桁落ちの本体）

- t は **必ず JulianDate2（2要素）から生成**する。`t = ((part1 − 2451545.0) + part2) / 36525`（centuries; millennia は /365250）。
  - `part1` は整数日（または J2000 相対整数日）に保ち、エポック 2451545.0 の減算を**整数部側で厳密**に行う。単一 f64 の JD（≈2.45e6）で減算すると ulp ≈ 5.4e-10 day ≈ **4.6e-5 s** を失い、±1s 目標を直接侵食する。
- JulianDate2 正規化規約: `part2 ∈ [-0.5, 0.5)`（または [0,1)）に保ち、時間オフセット加算（光行時間 τ 等）は `part2` に足して都度再正規化（§A3）。規約は conventions §6 と一致させる。

## A2. 速度の取得方法

用途ごとに必要精度が違う。バジェットは光行時間+光行差で 0.10″ 共有。

### (1) 年周光行差用：地球重心速度
- **解析微分（VSOP87 級数の項別微分）で取得**。`d/dt[A cos(B+Ct)] = −A·C·sin(B+Ct)` と自明・厳密・ほぼ無コスト。
- 必要精度: 光行差 = v/c。0.05″（=2.42e-7 rad）確保に **δv < 2.42e-7·c ≈ 73 m/s**。解析微分はこれを桁で上回る。
- Fast は簡易速度可。Standard/Reference は解析微分を既定。月（ELP）も同様に項別微分で速度供給。

### (2) 光行時間：位置反復（速度に依存しない）→ §A3
- 解析暦が高速なので、後退時刻での**位置再評価**で解く。速度外挿は使わない（堅牢性優先）。

### (3) ベッセル要素・接触の時間微分（dm/dt, ξ', η', x', y' …）
- 既定: **中心差分（2次, 5点で4次に昇格可）＋ Richardson 検証**。
- 刻み h: 固定値禁止。`h = c_h · T_char`（c_h ≈ eps^{1/3} ≈ 6e-6、T_char = 局所変動時間尺度。日食では時間〜時間スケール）。
  - 達成可能な相対誤差 ~ eps^{2/3} ≈ 3.7e-11（理論下限）。Besselian rate に対し過剰なほど良い。
  - **検証**: h と h/2 の差分を比較（Richardson）。一致が許容外なら h を調整、または解析微分に切替。これを受入テスト化（ISSUE-022/037）。
- 孤立した平滑スカラ関数で取りたい場合は **複素ステップ微分**（`Im[f(t+ih)]/h`, h~1e-20、減算的桁落ち無し）を選択可。ただし全チェーンを複素対応させる必要があるため、ephemeris 級数（解析微分で足りる）には用いない。

## A3. 光行時間反復

幾何位置 → 後退時刻 → 見かけ地心位置（conventions §5.1 の順序内）。

- **反復**: `τ_0 = 0; τ_{k+1} = |r_target(t − τ_k) − r_observer(t)| / c`（地心見かけは r_observer = 地心）。
- **収束基準**: `|τ_{k+1} − τ_k| < 1e-6 s`。
  - 根拠: 位置誤差 δr = v_target·δτ。月（v≈1.02 km/s, 距離384400km）で 0.05″ = 0.093 km → **δτ < 0.09 s** で足りる。1e-6 s は十分すぎる安全側。収束率 ~ v/c ~ 1e-4 で通常 **2–3 反復**。
- **上限**: 5 反復。未収束は `SolverDidNotConverge`（実運用では起きない安全弁）。
- 月の地心光行時間 ≈ 1.28 s、太陽 ≈ 499 s。いずれも同一反復器で処理（body 別の初期値不要）。
- 光速 c は conventions §4.1 の定義値（299792.458 km/s）。

## A4. ベッセル多項式フィット（経路/エクスポート用・022）

**局地の既定は直接評価（037, fit 誤差ゼロ, B2）**。多項式は**経路/GeoJSON/NASA 形式出力が本務**で、局地への転用は「**M2 で fit 残差 <0.01″（目標の 1/10）を実測実証後のバッチ局地（多地点）の任意最適化**」に限定し、**未達は直接（037）へフォールバック**（operations.md §D1）。実証前は局地で多項式を既定にしない。L7 で直接 vs 多項式残差を実測し局地転用の妥当性を裏取りする。

- **基底**: 正規化時刻 `τ = (t − t0)/Δ ∈ [−1, 1]` 上の **Chebyshev 最小二乗フィット**。
  - 単項式（Vandermonde）直接フィットは悪条件。Chebyshev で条件数を抑え、出力時に NASA 互換の単項式へ変換（D5: μ は度・hour 単位の対応表を付す）。
- **サンプル**: 瞬時要素（021）を **Chebyshev ノード**（または密な一様＋LS）で評価。Runge 現象を回避。
- **次数選択**: 低次（NASA 慣習の 3 次）から開始し、**fit 残差 `BesselFitError` < フィット配分（x,y で <0.10″ 相当）** を満たす最小次数を採用。最大次数を上限（過剰適合/Runge 防止、要確認: 上限値は M2 実測）。
- **フィット区間 Δ**: 中心食前後（おおむね P1..P4 か少し広め）。区間端での発散を残差ゲートで検出。
- **保持**: 採用基底・次数・区間・残差（BesselFitError）を結果に格納。直接(037) vs 多項式(022) の残差比較を L7 サブテスト化（accuracy §3.2）。

## A5. 求根・最小化・クランプ（横断）

- **求根（接触 C1–C4・合・地平線通過）**: Brent。要ブラケット（粗走査で符号変化を検出）。Newton 単独禁止（conventions §11）。
  - `root_tolerance_seconds` 既定 = **0.01 s**（**本数値が正本**: 全 docs はこの値に統一する。algorithms 各節の「≤0.2s」等の旧表記は本値へ修正済み）。根拠: solver 配分 0.05″ ≈ 0.1 s、その 1/10。EngineConfig 既定に設定。
- **最大食**: `dm/dt = 0` の Brent 求根（正式手法, D2）。最小化対象は **m² = u² + v²**（中心線尖点での √ 不連続回避）。距離最小化（黄金分割/009）は粗ブラケット用に限定。皆既帯平底の退行を fixture 化（accuracy §2.1）。
- **acos / asin クランプ**: 引数を `[-1, 1]` に丸め誤差込みでクランプ（食面積・角距離・基底構築）。境界（離隔/内包/外接/内接/同半径）を明示分岐（accuracy §2.2 / ISSUE-027）。
- **角度差の連続化**: 合・ゼロ点探索で ±π 折返しを除いた連続関数に変換してから解く（conventions §2）。

## A6. テスト・検証への落とし込み

- L1（純数学）: 級数和の丸め境界、JulianDate2 エポック減算の桁落ち回帰、中心差分の Richardson 一致、Brent/Chebyshev 単体。
- 各暦/幾何 Issue: 光行時間反復回数・収束、解析微分 vs 中心差分の一致、fit 残差ゲート。
- 既定閾値（収束 1e-6 s, root 0.01 s, h ≈ eps^{1/3}·T）はバジェット由来でコメント必須。**通すための閾値拡大は禁止**（conventions §11）。
